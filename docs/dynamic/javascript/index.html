Javascript Basics

  • events
  • alerts
  • functions

We can write javascript into an HTML document by using the <script> tag. Generally, we'll try and keep all of our javascript in the header of the document. Ideally, we'd keep the script out of the document all together by creating a new javascript document for it, and then linking that document to the HTML page in the header: <script type="text/javascript" language="javascript" src="scripts/ascript.js"></script>

Like in Actionscript, events are the things happening to an object: a click, the mouse moving over it, etc. Typically, we use events to call a function. We can put some javascript on an object receiving the events to call a function when they happen. This means putting some javascript into the body of your HTML code, which generally we will try and avoid doing.

'Alerts' are those message boxes which your browser uses to communicate with you. Generally, they're kind of annoying as they halt the progress of the page and wait for the user to click 'ok'. When writing a javascript, however, alerts are a great way of telling up what's going on.

Functions are the building blocks of code that you'll write to carry out particular tasks. Again, the syntax should be familiar from Actionscript:

function myfunction(){
alert("Hello world.");
}

We could call this function from a link by adding an 'onclick' attribute to it:

<a href="#" onclick="myfunction()">click for alert</a>

Which would do this: Click for alert

We're going to look at how to use javascript in conjunction with the document object model (DOM) and CSS to make some dynamic bits of HtML.
